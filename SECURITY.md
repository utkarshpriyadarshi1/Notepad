# Security Policy 🛡️

We take the security of **Notepad** seriously. If you believe you have found a security vulnerability in our application, please report it to us as detailed below.

---

## Supported Versions

Currently, security updates are applied to the active master branch and official minor releases.

| Version | Supported |
| ------- | --------- |
| >= 1.0.4| Yes       |
| < 1.0.4 | No        |

---

## Reporting a Vulnerability

**Please do not open a public GitHub issue for security-related bugs.**

Instead, report vulnerabilities privately by emailing us at **security@notepad-desktop.org**.

When reporting, please include:
- A detailed description of the vulnerability.
- Steps to reproduce (including a proof-of-concept payload or script if applicable).
- Impact assessment (e.g. data leak, local privilege escalation).
- Your operating system environment details.

We will acknowledge receipt of your report within 48 hours and work with you to analyze and resolve the issue.

---

## Disclosure Policy

- We will request up to 90 days to issue a fix before the vulnerability is publicly disclosed, in alignment with responsible disclosure principles.
- Once a fix is verified and merged, we will coordinate with you to publish a security advisory alongside a minor release update.
